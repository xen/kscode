  Аннотация: 
  
    Продукт sqlindex позволяет использовать в качестве
    индекса внешний поисковый инжайн - любую SQL-базу, к которой
    существует коннектор.

  Идея реализации:

    Продукт SQLIndex использует для обращения к базе данных два SQL-метода,
    один для добавления объекта и один для поиска. SQLIndex представляет
    собой контейнер, в котором и создаются эти два метода, под определенными
    именами -- например, select & update. Алгоритм индексации такой:
        
        -- Получить объект (который, очевидно, уже приведен к определненному
            интерфейсу);
            
        -- Из этого интерфейса получить список полей (есть такой вызов);
        
        -- За значением каждого поля обратится в переданный интерфейс;

        -- Сформировать словарь, как имя --> значение;
        
        -- Вызвать метод update;
        
    Алгоритмы запроса, удаления и переиндексации аналогичны. Количество Z
    SQL методов может быть расширено в соответствии с потребностями,
    опредяляемыми жизненным циклом мндексируемого объекта (насколько я
    помню, сейчас обрабатываются вызовы index, reindex, delete, request --
    для каждого из которых, очевидно, используется свой метод).

    Класс SQLIndex наследует от KeyIndexBase, что бы получать объект уже приведенный к индексируемому
    интерфейсу.

  Интерфейсы:
  
    ISQLIndex -- список параметров, позволяющих подрубится к SQL-серверу (может не потребоваться,
        если Z SQL Methods будут использовать свои собственнные настройки);
        
    ISQLContainer -- контейнер, содержащий Z SQL Methods.
