План тестирование трех видов поиска на RFC архиве:

  1. Пользуемся контент-типом Новость в качестве статьи

  2. Оформляем в виде трех юнит-тестов для каждого вида поиска

  3. Алгоритм теста для каждого вида поиска:

    1. Подготовка:

       - Очищается каталог.

    2. Индексирование:

       - Из базы RFC создаются объекты Новость для каждой статьи RFC.

    3. Переиндексирование:

       - Обновление всех созданных новостей, вследствие чего происходит переиндексирование

    4. Поиск:

       - Поиск по трем запросам, одинаковым по смыслу для всех трех видов поиска

    5. Очистка Индекса:

       - Удаление всех созданных новостей

    По пунктам 2-3 создаются тест-методы и замеряется время.
