  Аннотация: 
  
    Дать возможность отображать контент-классы на ресурсы.
    Контент-класс - это имя класса. Ресурс - некая параметризация
    этого класса. Для каждого класса существует одинаковый набор ресурсов.

  Идея реализации:

    Создаем контейнер. В контейнере делаем форму добавления,
    позволяющую в качестве имени контента указать
    имя класса в дот-нотации. Имена известных классов можно дернуть
    из Utilities(IFactories). Сам контент - это список ресурсов.
    
    Имеет интерфейс IContentClassResource, и содержит
    кучу атрибутов, по одному для каждого ресурса.
    
    Примерный список ресурсов:
    
        call -- Русское название класса,
         
        icon -- Икона класса.
        

    При инстанциировании утилиты, для каждого ресурса создается и
    регистрируется словарь IVocabulary, имя которого собирается из
    префикса и имени ресурса. Словарь задает отображение дот-имени
    класса на конкретный ресурс.
    
    Список словарей собирается по интерфейсу IContentClassResource.
  
 Виды:
 
    edit.html -- редактирования файла ресурсов,
    
    contents.html -- редактирование списка ресурсов,
    
    add_resource -- добавление ресурса.
    
 Состав:
 
    ResourceContainer 
    
    ResourceContent
    
    ClassVocabulary -- словарь списка классов для выбора имени в форме
        добавления.
