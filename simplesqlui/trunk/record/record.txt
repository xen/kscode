  Аннотация: 
  
    Класс record -- базовый класс, аналог класса persistent, позволяет
    прозрачно получать и модифицировать атрибуты объекта, сохраняя
    их в базе данных.

  Идея реализации:

    Каждый атрибут объекта, хранимый в базе данных,
    задается специальным классов, аналогичным property.
    
    Полный список значений атрибутов объекта, хранится 
    в словаре __dict_. Обращение к атрибуту на чтение прочитывает
    атрибут из словаря, а при его отсутствии - вызывает
    метод select источника данных, из атрибута __ds_ объекта.
    
    Обращение к атрибуту на запись вызывает модификацию атрибута
    в словаре и регистрацию объекта в менеджере транзакций. При
    закрытии транзакции, будет вызван метод update источника
    данных и сохранит все атрибуты словаря (в будущем возможно
    избирвательное сохранения изменившихся атрибутов).

    Инициализация объекта record включает в себя передачу следующих
    аргументов: 
    
        __parent__ -- ссылка на родителя,
        
        ds -- источник данных для данного объекта,
        
        ids -- кортеж идентификаторов, используемых для идентификации
            объекта.
    
    Существует особый класс атрибутов - связи между объектами.
    Обращение к такому атрибуту позволяет получить список объектов,
    связанных с данным, при помощи специальной фабрики.
    
    Сам по себе этот класс атрибутов не используется, а вызывается
    интерфейс контейнера, ключи которого строятся по следующему принципу::
    
        <КЛЮЧ> :== <СВЯЗЬ> ["." <ID> [...] ]
        
        <СВЯЗЬ> :== наименование атрибута;
        
        <ID> :== один из идентификаторов объекта; 
    
    В реальной жизни интерфейс контейнера и его виды не используется, так 
    как неудобны (недостаточны) при работе с базами данных.
    
    Используются производные от него интерфейсы ISQLContainer, каждый из
    которых содержит один атрибут связи, а использующий его вид
    показывает только объекты, ограниченные полученными по этой связи.

  Состав:
  
    RecordBase -- базовый класс;
    
    sqlattribute -- конструктор атрибута;
    
    sqljoin -- конструктор связи.
    
  Виды:
  
    В первой версии разрабатывается только упрощенный
    вид ISqlContainer или не разрабатывается вообще (используется стандартный
    вид просмотра контейнеров);
